## Chat

### Live model and multiplier details for Copilot CLI and Claude agent

**Setting**: `setting(github.copilot.chat.agent.modelDetails.enabled)`

[Copilot CLI](https://code.visualstudio.com/docs/copilot/agents/copilot-cli) and [Claude agent](https://code.visualstudio.com/docs/copilot/agents/claude-agent) responses in the Chat view show the last used model and the model multiplier on each response, so you can see at a glance which model handled each turn and how it counts against your usage. The badge appears live as each response completes, without needing to reload the window, and updates when you switch models mid-session.

When using **Auto** model selection in Copilot CLI, the badge displays the actual resolved model (for example, `Claude Sonnet 4.6`) rather than `auto`. The resolved model is also preserved when the session is rebuilt from history.

The behavior is enabled by default. To turn off the badge, disable the `setting(github.copilot.chat.agent.modelDetails.enabled)` setting and reload the window.

### Lightweight background agent to manage Todo Lists (Experimental)

During multi-step agent tasks, keeping the todo list in sync with actual progress relies on the main model deciding to update it using tool calls. To still maintain the user experience of showing model progress using a todo list, but reducing token usage for the main agent using a frontier model this experiment uses a background agent to manage the todo list.

When this setting is enabled, a lightweight background agent with a smaller model monitors main agent activity and updates the todo list to reflect completed and in-progress work when necessary. The main agent will not have the todo tool available, thus saving token cost for the conversations.

> Note: if the todo tool is manually added to the chat request (eg: with `#todo`), or a custom agent specificies it in its tool list, the background agent will be overidden and not run.

This feature is disabled by default. To try it out, enable the `setting(github.copilot.chat.agent.backgroundTodoAgent.enabled)` setting.
