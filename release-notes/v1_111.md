## Agent-scoped chat hooks (Preview)

Custom agent frontmatter now supports agent-scoped chat hooks. You can define hooks directly in the YAML frontmatter of your `.agent.md` files, and they only run when the custom agent is selected or invoked via `runSubagent`. This lets you attach pre- and post-processing logic to specific agents without affecting other chat interactions.

To try this feature, enable the `setting(chat.useCustomAgentHooks)` setting. For more information on the hooks format, see the [chat hooks documentation](https://aka.ms/vscode-chat-hooks).

## Chat tip improvements

Chat tips now guide you through a structured onboarding journey. Foundational tips, such as selecting Plan and creating a custom agent are shown first. After you complete or dismiss the foundational tips, quality-of-life tips like experimental settings or generating Mermaid diagrams are shown in random order.

Additional chat tip improvements include:

* Tips are only displayed when a single chat session is visible, such as in the welcome view or the panel chat. If multiple chat editors are open, tips are hidden to reduce clutter.
* Tips can now include keyboard shortcuts to help you discover relevant key bindings.
* New tips were added for the `/init` and `/fork` slash commands. The `/init` tip helps you discover the command for initializing project configuration, and the `/fork` tip introduces manual conversation forking, which lets you branch a conversation to explore different approaches.
* Tips are hidden after you action or dismiss them for the current session.

## Debug events snapshot

You can now attach a snapshot of agent debug events as context in chat by using `#debugEventsSnapshot`. Use it to ask the agent about loaded customizations, token consumption, or to troubleshoot agent behavior.

You can also select the sparkle chat icon in the top-right corner of the Chat Debug panel to add the debug events snapshot as an attachment to the chat composer. Selecting the attachment opens the Chat Debug panel logs, filtered to the timestamp when the snapshot was taken.

## AI CLI profile group in terminal dropdown (Experimental)

With `setting(terminal.integrated.experimental.aiProfileGrouping):true`, AI CLI terminal profiles, such as GitHub Copilot CLI, are now displayed in a dedicated group at the top of the terminal profile dropdown for improved discoverability.

## Engineering

### Test plan item creation button

A button was added to streamline creating test plan items directly from feature request issues. This reduces the manual steps needed to set up structured testing plans for new features.

### Add verification steps button

A button was added to generate verification steps on issues that need them. This helps ensure that issues have clear, structured steps for verifying fixes and features before they are closed.

### Chat showcase pipeline

A new automated pipeline processes issues labeled `chat-showcase`. When a showcase issue is identified, a corresponding chat tip issue is automatically created, making it easier to add tips for features.

### Automatic PR media attachment to linked issues

When you merge a pull request that includes an image or GIF in its description, the media is now automatically posted as a comment on the linked issue. This streamlines the verification process by making it easier to see the visual demonstration of a fix or feature directly on the issue.

### Upcoming deprecations

* **Edit Mode** is officially deprecated as of VS Code version 1.110. Users can temporarily re-enable Edit Mode via VS Code setting `setting(chat.editMode.hidden)`. This setting will remain supported through version 1.125. Beginning with version 1.125, Edit Mode will be fully removed and can no longer be enabled via settings.
