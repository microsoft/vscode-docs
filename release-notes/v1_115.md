## Editor Experience

### Integrated browser tool improvements

**Setting**: `setting(workbench.browser.enableChatTools)`

#### Better tool labels

Browser tool calls now have more descriptive labels and link directly to the target tab.

_Old:_<br/>
![A tool call saying "Clicked element in browser"](images/1_115/browser-tool-old.png)

_New:_<br/>
![A tool call saying "Right-clicked header banner in Test Page", with a link to Test Page](images/1_115/browser-tool-new.png)

#### Long-running script support

The `Run Playwright Code` tool has improved support for long-running scripts. Scripts that take longer than 5 seconds to run (by default) will now return a deferred result for the agent to poll.

#### Fewer duplicate tabs

Agents are now more heavily discouraged from opening repetitive browser tabs. Now, when an agent attempts to open a new tab and an available tab is already open to the same host, no new tab will be opened unless an explicit flag is passed by the agent.

### Pinch-to-zoom in the integrated browser (macOS)

The [integrated browser](https://code.visualstudio.com/docs/debugtest/integrated-browser) now supports pinch-to-zoom on macOS. Use the trackpad pinch gesture to magnify web page content up to 3x.

Unlike the standard browser zoom (`kb(workbench.action.browser.zoomIn)` / `kb(workbench.action.browser.zoomOut)`), pinch-to-zoom is a purely visual magnification and doesn't reflow the page layout.

## Terminal

### Background terminal notifications

Previously, when a terminal command was running in the background, the agent had to manually call `get_terminal_output` to check on its status, without knowing when the command completed or needed input.

With the new experimental `setting(chat.tools.terminal.backgroundNotifications)` setting, the agent is automatically notified when a background terminal command finishes or requires user input. This also applies to foreground terminals that time out and are moved to the background. The agent can then take appropriate action, such as reviewing the output or providing input via the `send_to_terminal` tool.

### Send input to background terminals

A new `send_to_terminal` tool enables the agent to send text and commands to background terminals. Previously, background terminals were read-only, with only `get_terminal_output` available. This was particularly limiting when a foreground terminal timed out and moved to the background, as the agent could no longer interact with it.

With `send_to_terminal`, the agent can continue interacting with background terminals. For example, if an SSH session times out while waiting for a password prompt, the agent can still send the required input to complete the connection.
