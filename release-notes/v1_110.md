## Editor Experience

### Modal Editors (Settings, Keybindings,...)

**Setting**: `setting(workbench.editor.useModal)`, `setting(extensions.allowOpenInModalEditor)`

We are experimenting with a new modal editor experience for certain editors that you are typically going in to adjust something and go out again. This applies to:
* Settings
* Keyboard shortcuts
* Profiles management
* AI and Language models managment
* Workspace trust management

Configuring `setting(workbench.editor.useModal)` to `some` opts into this experience:

![Screenshot showing a modal settings editor.](images/1_110/modal-editors.png)

A modal editor floats on top of the editor and quickly closes when you press Escape key. There is an action to move the editor back into the main area as well as an action to maximise the modal experience.

> **Note**: settings and keybindings editor show a button to open the associated JSON file as text editor

Another setting `setting(extensions.allowOpenInModalEditor)` expands the use of modal editors to extensions. This experience is still work in progress and will likely change in the future but we wanted to make it available for feedback now:

![Screenshot showing a modal extensions editor.](images/1_110/modal-extensions.png)

In this modal editor a control in the title bar allows to navigate between all extensions from the list you had in the extensions sidebar. The same applies to MCP servers as well. Future versions will likely see the extensions list and search functionality move into the modal too. We are grateful for feedback from this experience, so please try it out and let us know what you think.
