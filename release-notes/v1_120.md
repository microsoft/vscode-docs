## Agents

### Agents window

We've found ourselves and our users working with AI in different ways depending on the task at hand. Some workflows are code-first: writing code in the editor and using AI as a coding assistant. An increasing share of workflows are agent-first: handing off higher-level tasks to an agent and focusing on outputs and orchestration.

As agents take on a growing share of work, it's valuable to have the option to shift fully into an agent-first environment. The **Visual Studio Code Agents** window is our dedicated experience built for the agent-first way of working. You can use it to run sessions in parallel across multiple workspaces without opening a separate VS Code window for every project.

The Agents window has been available as part of VS Code Insiders in our past few releases, and with this release, it's now available with VS Code Stable. The window can be opened in several ways, including the "Open in Agents" button in the titlebar of VS Code. You can freely move between VS Code and the Agents window as you work on different tasks or as your workflows evolve.

<!-- TODO: embed latest Agents window video here -->

We appreciate the user feedback we've received while shaping Agents, and this release addresses several of the most frequent requests:

* **Picker choices persist across new sessions**: Your last choices in pickers like agent harness and isolation mode are remembered when you create new sessions.
* **Discard changes more easily**: You can discard edits directly from the Changes panel.
* **Sync upstream changes in new sessions**: A sync button on the Files panel lets you see upstream changes from the base branch and pull them in before the agent gets to work.
* **More deterministic Changes interactions**: Actions in the Changes panel can complete more quickly as they're now deterministic.
* **View All Changes by default for completed sessions**: When you open a session marked as done, you automatically get a view of the agent's full set of edits at a glance.

You can review more information about how to get started with and use the Agents window in its [documentation](https://aka.ms/VSCode/Agents/docs).

Your feedback continues to be a great help in shaping Agents. Please continue to [file issues on GitHub](https://github.com/microsoft/vscode/issues) or feel free to browse [existing issues](https://github.com/microsoft/vscode/issues?q=state%3Aopen%20label%3A%22agents-window%22).
