## Language Models

### Install model providers from the Language Models editor

With Bring Your Own Key (BYOK) models, the extension ecosystem expands this further with extensions that contribute their own model providers. Previously, discovering these extensions meant knowing to search the Extensions view for the right tag. The Language Models editor includes an **Install Model Providers** button that takes you straight to the relevant extensions in the Marketplace.

To use it, open the Language Models editor (**Chat: Manage Language Models** command) and select **Install Model Providers**. VS Code opens the Extensions view, filtered to language model provider extensions, where you can install a provider and follow its setup steps to configure additional models. After installation, the extension's models appear in the model picker alongside your other models.

<video src="images/1_125/install_model_providers.mp4" title="Video showing the Install Model Providers button in the Language Models editor opening the Extensions view filtered to model provider extensions." autoplay controls muted></video>

To learn more, see the [language models documentation](https://code.visualstudio.com/docs/agent-customization/language-models).

### Integrated Browser

#### Web Search from Address Bar

Setting: `setting(workbench.browser.searchEngine)`

You can now search the web from the integrated browser's address bar, just like in a regular browser. Set `setting(workbench.browser.searchEngine)` to configure this feature.

#### Browse over remote connections (Preview)

Setting: `setting(workbench.browser.enableRemoteProxy)`

When the integrated browser is opened in a remote workspace, web traffic over HTTP(S) can now be proxied via the remote connection. This allows you to securely connect to any ports or services that are only accessible from the remote machine.

This is a preview feature -- you may encounter bugs. Enable the `setting(workbench.browser.enableRemoteProxy)` setting to try it out, and please file any issues you encounter.

#### Better agentic interaction with forwarded ports

If you have forwarded a port in a remote workspace, previously agents could have difficulty opening the browser due to a potentially different port number.

Now, if an agent requests a port that has been forwarded (and the remote proxy is not enabled), the URL will be rewritten and the agent notified of the change.

## Enterprise

### Native MDM delivery for managed Copilot settings

Organizations can now deliver managed Copilot settings through native device management (MDM) in addition to the account-based enterprise settings file, building on the [enterprise-managed Copilot plugin policies](https://code.visualstudio.com/updates/v1_124#_enterprise).

Future updates will expand the set of supported policy keys across Copilot surfaces.
